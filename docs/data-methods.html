What do CRAN download counts mean? [probably need a catchier title than this]

Section 3 Data & Methods

The main source of data used in this report is the download logs from the RStudio CRAN mirror site https://cran.rstudio.com/.

In this project, we extracted R package names from CRAN and got summary daily download logs of R packages through web API maintained by r-hub(R-Hub, n.d.). And we also collected release dates of R packages through CRAN. In addition, we also tried to scrap the number of commits in Github repository for R packages. Then we constructed eight exploratory data analysis sections:

    1. we explored the daily download of all the packages on CRAN and figured out the number of packages occupied by different downloaded groups;
    1. we analyzed the daily download trend of R itself and the found out the most popular version of R;
    1. we compared the top 15 downloaded packages on CRAN on 1st April. from year 2013 to 2021 to see how the user preference changes;
    1. we figured out the relationship between initial release date and the total download count for last half a year;
    1. we compared two closely related packages fable and forecast to see the difference between their MA (moving average);
    1. we studied how the number of commits of master (main) branch on Github repository influence the total download count for last half a year;
    1. we explored the relationship between the number of updates of packages and its download count;
    1. we analyzed the name length pattern for both CRAN task view packages and all the packages on CRAN;
    1. we explored how the alphabetical order of packages affected the download count and its statistic features (variance, median).
